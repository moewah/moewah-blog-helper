# 文章标点符号规范

适用场景：包含代码块、中英文混排的技术分享类文章。

## 1. 全角与半角

| 场景 | 标点类型 |
|------|----------|
| 中文叙述 | 全角标点（，。：；“ ” ‘ ’ ！？） |
| 英文、数字、代码、行内代码 | 半角字符 |
| 中文字符与英文/数字/行内代码之间 | **加一个空格** |

示例：
- 正确：使用 Docker 部署应用非常方便。
- 错误：使用Docker部署应用非常方便。

## 2. 行内代码

- 前后加空格
- 标点（全角）放在反引号外面

示例：
```
正确：请使用 `npm install` 命令安装依赖。
错误：请使用`npm install`命令安装依赖。
错误：请使用 `npm install`命令安装依赖。
```

## 3. 代码块

- 代码块内部保留原始编程语言格式，不添加任何中文标点
- 代码块前使用中文全角冒号引出
- 代码块后接完整中文句子（以句号等结束）

示例：
````
代码如下：

```python
def hello():
    print("Hello")
```

上述代码定义了一个函数。
````

## 4. 括号与引号

| 场景 | 使用类型 |
|------|----------|
| 中文叙述中的括号、引号 | 全角（“ ” ‘ ’ （ ）） |
| 代码上下文或纯英文短句 | 半角 |

示例：
- 正确：这个功能被称为"热更新"（Hot Reload）。
- 正确：The function `useState()` returns an array.

## 5. 列表与省略号

**列表项**：

- 若为完整句子，句末加全角句号
- 若为短语，保持统一不加标点

**省略号**：使用全角"……"

示例：
```
正确的列表：
- 配置环境变量。
- 安装依赖包。
- 启动服务。

或（短语形式）：
- 环境变量配置
- 依赖包安装
- 服务启动
```

## 6. 一致性

全文保持标点风格统一，中英文混排处空格一致。

## 检查清单

处理文章时，按以下顺序检查：

1. [ ] 中文叙述是否使用全角标点
2. [ ] 中英文/数字/行内代码之间是否有空格
3. [ ] 行内代码前后是否有空格，标点是否在外
4. [ ] 代码块内部是否保持原始格式
5. [ ] 括号引号是否按场景使用全角/半角
6. [ ] 列表项标点是否统一
7. [ ] 全文风格是否一致
